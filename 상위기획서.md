# 인스타그램 릴스 자동 생성 프로그램 - 상위 기획서

## 1. 프로젝트 개요

### 1.1 목적
최신 트렌드 키워드를 입력받아 관련 콘텐츠와 이미지를 자동으로 수집하고, 이를 바탕으로 인스타그램 릴스 영상을 자동 생성하는 프로그램 개발

### 1.2 주요 기능
- 키워드 기반 콘텐츠 검색 및 수집
- 이미지/영상 소스 자동 수집
- 텍스트-음성 변환 (TTS)
- 자동 영상 편집 및 합성
- 릴스 포맷 최적화 (9:16 세로 영상, 15-90초)

## 2. 기술 스택 제안

### 2.1 콘텐츠 수집
- **텍스트 콘텐츠 수집**
  - OpenAI GPT API: 키워드 기반 대본 생성
  - Google News API / Naver News API: 최신 뉴스 수집
  - Twitter/X API: 실시간 트렌드 파악
  - Reddit API: 커뮤니티 반응 수집

- **이미지/영상 수집**
  - Unsplash API: 무료 고품질 이미지
  - Pexels API: 무료 영상 클립
  - Pixabay API: 무료 이미지/영상
  - DALL-E / Midjourney API: AI 이미지 생성 (옵션)

### 2.2 영상 생성
- **Python 기반**
  - MoviePy: 영상 편집 및 합성
  - Pillow (PIL): 이미지 처리
  - FFmpeg: 영상 인코딩/변환
  - OpenCV: 고급 영상 처리

### 2.3 텍스트-음성 변환 (TTS)
- ElevenLabs API: 자연스러운 음성 생성
- Google Cloud TTS: 다국어 지원
- OpenAI TTS: 품질 좋은 음성
- Naver Clova Voice: 한국어 특화

### 2.4 자막 생성
- AssemblyAI: 음성-텍스트 변환 (선택)
- 자체 타이밍 계산 알고리즘
- 텍스트 애니메이션 효과

## 3. 시스템 아키텍처

```
[사용자 입력: 키워드]
        ↓
[콘텐츠 수집 모듈]
    ├─ 텍스트 생성 (GPT)
    ├─ 이미지 검색 (Unsplash, Pexels)
    └─ 트렌드 분석 (News API)
        ↓
[콘텐츠 처리 모듈]
    ├─ 대본 최적화 (15-60초 분량)
    ├─ 이미지 선별 및 다운로드
    └─ 음성 생성 (TTS)
        ↓
[영상 제작 모듈]
    ├─ 이미지 시퀀스 생성
    ├─ 전환 효과 적용
    ├─ 자막 오버레이
    ├─ 배경음악 추가
    └─ 음성 합성
        ↓
[후처리 모듈]
    ├─ 릴스 포맷 최적화 (1080x1920)
    ├─ 품질 검증
    └─ 메타데이터 추가
        ↓
[최종 출력: MP4 파일]
```

## 4. 주요 기능 명세

### 4.1 입력
- **필수 입력**
  - 주제 키워드 (예: "AI 트렌드", "건강한 다이어트")
  
- **선택 입력**
  - 영상 길이 (15초, 30초, 60초, 90초)
  - 스타일 선택 (뉴스형, 교육형, 엔터테인먼트형)
  - 음성 톤 (남성/여성, 감정 톤)
  - 배경음악 장르
  - 자막 스타일

### 4.2 처리 과정

#### Step 1: 콘텐츠 기획
1. 키워드 분석 및 관련 검색어 확장
2. 최신 트렌드 데이터 수집
3. GPT를 활용한 대본 생성 (훅 - 본문 - 결론 구조)
4. 장면별 필요 이미지 키워드 추출

#### Step 2: 소스 수집
1. 각 장면에 맞는 이미지/영상 클립 검색
2. 저작권 확인 (API 라이선스 정보 체크)
3. 이미지 품질 필터링 (해상도, 적합성)
4. 로컬 저장소에 다운로드

#### Step 3: 음성 생성
1. 대본을 문장 단위로 분할
2. TTS API를 통해 음성 파일 생성
3. 음성 길이에 맞춰 이미지 노출 시간 계산

#### Step 4: 영상 합성
1. 이미지를 시간순으로 배치
2. Ken Burns 효과 (줌인/아웃, 패닝) 적용
3. 장면 전환 효과 추가
4. 자막 타이밍 동기화
5. 배경음악 삽입 (음성과 볼륨 밸런싱)
6. 최종 렌더링 (1080x1920, 30fps, H.264)

### 4.3 출력
- **파일 형식**: MP4 (H.264 코덱)
- **해상도**: 1080 x 1920 (9:16 비율)
- **프레임레이트**: 30fps
- **비트레이트**: 5-8 Mbps
- **오디오**: AAC, 128kbps
- **부가 파일**: 
  - 썸네일 이미지 (자동 생성)
  - 추천 해시태그 목록
  - 대본 텍스트 파일

## 6. 기술적 고려사항

### 6.1 성능
- 영상 1개 생성 시간 목표: 2-5분
- 병렬 처리를 통한 다중 영상 제작
- 캐싱 전략 (자주 사용되는 이미지, 음악)

### 6.2 비용
- API 호출 비용 산정
  - GPT-4: 대본 생성당 $0.01-0.05
  - TTS: 음성당 $0.01-0.03
  - 이미지 API: 대부분 무료 (일일 제한 있음)
- 서버 비용 (영상 렌더링용)

### 6.3 저작권
- 라이선스가 명확한 소스만 사용
- API 이용 약관 준수
- 상업적 이용 가능 여부 확인

### 6.4 품질 관리
- 부적절한 이미지 필터링
- 대본 검수 로직 (욕설, 민감 주제)
- 음성-자막 싱크 검증
- 영상 품질 자동 체크

## 7. 예상 구현 시간

## 8. 리스크 및 대응방안

### 8.1 기술적 리스크
- **API 장애**: 대체 API 준비, 폴백 메커니즘
- **렌더링 성능**: GPU 가속, 클라우드 렌더링 고려
- **저작권 이슈**: 라이선스 명확한 소스만 사용

- **인스타그램 정책 변경**: API 없이도 작동하는 구조
- **콘텐츠 품질**: 사람의 최종 검수 단계 추가
- **비용 증가**: 프리티어 우선 사용, 비용 모니터링
